## 关键修订点

1. 模块一的新建清洗流程页面现在突出“选择数据表、填写 `process(df)` 核心代码、运行当前流程、保存可复用流程、批量应用到多张数据表”。第一版采用受控流程处理器生成标准结果，学生填写的 Python 代码用于流程定义、规范表达和复用登记，不在现场直接执行任意代码。
2. 模块一的数据质量报告保存到 `outputs/reports/data_quality_report.csv`，不再和清洗数据混在同一目录。系统启动时会清理上一轮质量报告，避免评委看到旧报告。
3. 模块一的数据连接模块只做一个主表、多个连接表、一个连接字段的操作界面。现场建议主表选 `products.csv`，连接表选 `orders.csv`、`inventory.csv`、`returns.csv`，连接字段选 `product_id`，输出 `data/cleaned/product_base_metrics.csv`。
4. 模块二严格使用 `product_base_metrics.csv` 作为输入，ABC 分类在模块二完成，模块一不生成 `abc_class`。
5. 模块三异常订单诊断以当前系统为准：运行后展示异常配送订单约 312 条，其中 A类重点商品超时 46 条、生鲜冷链超时 28 条、超长配送 96 条、乡镇偏远超时 72 条。不要继续讲 684 条异常订单。
6. 模块三“网点异常汇总与风险评分”依赖上一页输出的 `outlet_exception_summary.csv`，再合并模块一的 `outlet_metrics.csv`，风险权重由异常订单结果 60% 和网点基础数据 40% 组成。
7. 模块三 K-Means 的不同颜色表示不同配送优化片区，不表示风险等级。现场推荐选择 K=4，对应地图上四个空间片区。
8. 模块三取消“起降点方案对比与结果交接”讲法。当前流程是：K-Means 聚类后筛选起降点候选点，再直接进入 AI低空应急配送航线智能规划系统。
9. 模块三起降点候选页现场选择“每个聚类片区候选点数量 = 2”，结果保证四个片区各 2 个候选点，共 8 个。
10. 模块三低空航线页要体现强化学习模拟配置：手动选择 State 状态变量、Action 动作空间、Reward 奖励函数权重，并设置 alpha、gamma、epsilon 和训练轮次。

# 二、模块一：学生1 数据工程师

系统入口：`streamlit run app.py`  
模块名称：可定制数据清洗流程中心  
建议时长：10-12 分钟

## 模块一句话定位

模块一负责把企业原始 CSV 数据加工为后续 AI 库存分析和配送规划可用的标准化数据资产。本模块只做数据导入、数据质量检查、通用清洗、批量清洗和数据连接，不做 ABC 分类。

## 页面与操作顺序

左侧菜单依次操作：

1. 模块首页
2. 数据源管理
3. 数据质量概览
4. 新建清洗流程
5. 数据连接模块

## 讲稿与操作

各位评委老师好，我是学生1，岗位是数据工程师。我的任务是把物流企业分散的原始数据整理成标准化数据，为后续库存分析、配送规划和经营报告提供数据基础。

### 第一步：进入模块首页

我首先进入“可定制数据清洗流程中心”。首页明确说明了本模块的边界：本模块负责数据导入、质量检查、清洗流程注册、批量清洗和基础指标连接，不做 ABC 分类，也不训练模型。ABC 分类会交给学生2在模块二完成。

### 第二步：数据源管理

现在我进入“数据源管理”页面，点击“刷新数据源”。

系统会扫描 `data/raw`、`data/cleaned` 和 `data/processed` 下的 CSV 文件，并显示每张表的文件名、业务类型、路径、行数、列数、缺失值数量和重复行数量。

本次模拟企业包含以下核心数据：

- `products.csv`：商品表，200 个商品。
- `orders.csv`：订单表，约 7926 条订单。
- `inventory.csv`：库存表，200 条商品库存。
- `returns.csv`：退货表，约 500 条退货。
- `delivery.csv`：配送表，约 7926 条配送记录。
- `outlets.csv`：网点表，60 个配送网点。

我可以选择任意文件预览前 20 行，确认字段是否读取正常。这里重点查看 `orders.csv`，因为订单表是后续销量统计、库存分析和经营指标生成的基础。

### 第三步：数据质量概览

接下来进入“数据质量概览”。这里可以选择不同数据表生成质量报告。

我选择 `orders.csv`，点击“生成数据质量报告”。

系统会生成 `outputs/reports/data_quality_report.csv`。这个报告只用于质量检查，不与清洗后的业务数据混放。每次系统启动时会清理上一轮质量报告，保证当前报告来自本次检查。

当前订单数据主要问题控制在通用清洗范围内：

- 重复订单，需要按 `order_id` 去重。
- `quantity` 字段存在空值，需要填补或复核。
- `order_date` 可能存在格式不统一，需要统一成标准日期格式。

这些问题属于多张业务表都可能遇到的通用数据质量问题，包括空值、重复值和字段格式问题。现场时间有限，所以本模块不追求复杂异常类型，而是先把通用清洗流程跑通。

### 第四步：新建清洗流程

现在进入“新建清洗流程”。这个页面是学生1的核心操作页面。

第一步，我先选择要处理的数据表，这里选择 `orders.csv`。页面会显示订单表的行数、列数、缺失值数量等基础信息。

第二步，我在“选择代码模板”中选择“通用数据清洗模板”，点击“载入模板”。系统会把模板函数填入 Python 核心流程代码框。

这里的代码框要求学生写的是以 DataFrame 为输入参数的 `process(df)` 函数。例如订单清洗可以表达为：

```python
def process(df):
    df = df.copy()
    df = df.drop_duplicates(subset=["order_id"])
    df["quantity"] = df["quantity"].fillna(1)
    df["order_date"] = pd.to_datetime(df["order_date"], errors="coerce").dt.strftime("%Y-%m-%d")
    return df
```

我在现场讲解时会强调：这段代码体现了数据工程师的处理思路，包括去重、空值填补和日期格式统一。当前第一版平台采用受控流程处理器生成标准化清洗结果，保证比赛演示稳定；学生填写的代码会作为流程定义保存，后续可以替换为真实执行引擎。

第三步，我点击“运行当前流程”。系统生成处理后结果，例如 `clean_orders.csv`，并显示处理后预览。这里可以对比处理前后行数、缺失值和重复值变化。

第四步，我把这个处理过程保存成可复用流程。填写流程名称、流程分类、输出结果名称、作者、版本号、流程说明和处理参数，点击“保存处理流程”。

第五步，我在“批量应用到其他数据”区域选择刚才注册的流程，再多选要清洗的数据表，例如 `orders.csv`、`products.csv`、`returns.csv`。点击“批量清洗选中数据”后，系统会将通用清洗规则应用到多张数据表，并把结果保存到 `data/cleaned` 目录。

这里的业务解释是：虽然我最先基于订单表编写清洗逻辑，但平台把流程沉淀为“字段识别 + 规则映射”的通用处理资产。复用到其他数据表时，系统会根据字段名称、数据类型、日期字段特征和重复规则，执行空值处理、去重和格式标准化。

### 第五步：数据连接模块

最后进入“数据连接模块”。这个页面用于生成模块二需要的商品基础指标表。

操作方式很简单：

1. 主表选择 `products.csv`。
2. 连接表多选 `orders.csv`、`inventory.csv`、`returns.csv`。
3. 连接字段选择 `product_id`。
4. 点击“生成连接结果”。

系统会自动完成以下连接逻辑：

- `products.csv` 作为商品主表，一行一个商品。
- `orders.csv` 按 `product_id` 汇总销量、销售额和出库频次。
- `inventory.csv` 按 `product_id` 汇总库存数量和库存金额。
- `returns.csv` 按 `product_id` 汇总退货次数和退货率。
- 最后按 `product_id` 合并生成 `product_base_metrics.csv`。

输出文件保存到 `data/cleaned/product_base_metrics.csv`。这张表包含商品名称、类别、销售指标、库存指标、退货指标和周转指标，但不包含 `abc_class` 字段。

### 模块一总结

我的模块完成了从原始数据到标准分析数据的转换。通过数据源管理、质量报告、清洗流程编写、流程注册、批量清洗和数据连接，系统生成了 `clean_orders.csv` 和 `product_base_metrics.csv` 等关键文件。模块一只负责打好数据基础，后续 ABC 分类、预测和库存决策由学生2继续完成。

# 三、模块二：学生2 AI库存分析工程师

系统入口：`streamlit run app2.py`  
模块名称：AI库存分析与ABC智能分类系统  
建议时长：10-15 分钟

## 模块二一句话定位

模块二基于模块一输出的 `product_base_metrics.csv`，完成 ABC 分类、销量预测、库存预警、补货清仓建议，并输出 `high_priority_products.csv` 给模块三使用。

## 页面与操作顺序

左侧菜单依次操作：

1. 库存分析数据接收
2. ABC分类提示词编写
3. 大模型代码生成演示
4. ABC分类代码运行与结果展示
5. 销量预测模型训练与评估
6. 补货清仓建议与结果交接

## 讲稿与操作

各位评委老师好，我是学生2，岗位是 AI库存分析工程师。我的任务是基于学生1交付的商品基础指标表，完成商品经营重要性分类、销量预测和库存决策建议。

### 第一步：库存分析数据接收

我首先进入“库存分析数据接收”页面。系统读取模块一生成的 `product_base_metrics.csv`。如果文件在 `data/cleaned` 目录，模块二也可以兼容读取。

页面会展示商品总数、商品类别、销售金额、库存金额、退货率等基础指标，并预览前 20 行数据。确认数据无误后，点击“确认接收数据”。

### 第二步：ABC分类提示词编写

接下来进入“ABC分类提示词编写”页面。

这里不是直接点一个按钮出结果，而是由学生编写提示词，让大模型生成 Pandas 分析代码。提示词需要明确输入文件、字段、分类规则和输出文件。

我输入的核心提示词是：

请你作为物流企业库存分析工程师，基于 `product_base_metrics.csv` 商品基础指标表，生成 Python Pandas 代码，用于完成商品 ABC 分类分析。要求按 `sales_amount_90d` 从高到低排序，计算累计销售贡献率 `cum_rate`，累计贡献率小于等于 0.7 为 A 类，大于 0.7 且小于等于 0.9 为 B 类，大于 0.9 为 C 类。输出 `abc_result.csv` 和 `abc_summary.csv`，并添加中文注释，便于现场讲解。

我点击“提交给大模型生成代码”，进入下一页。

### 第三步：大模型代码生成演示

进入“大模型代码生成演示”页面后，我点击“开始生成ABC分类代码”。

系统展示大模型生成代码的过程，包括读取 CSV、按销售额排序、计算累计贡献率、划分 A/B/C 类、保存结果文件等步骤。

然后点击“保存为ABC分类分析脚本”。这一步体现了大模型辅助学生生成数据分析代码的能力。

### 第四步：ABC分类代码运行与结果展示

接下来进入“ABC分类代码运行与结果展示”，点击“运行生成代码”。

系统生成：

- `abc_result.csv`：ABC 分类明细表。
- `abc_summary.csv`：ABC 分类汇总表。

页面会展示 A/B/C 商品数量、销售贡献占比、库存金额占比和累计销售贡献曲线。

这里的业务解读是：

- A类商品销售贡献最高，是企业应优先保障的重点商品。
- B类商品保持正常库存和采购节奏。
- C类商品销售贡献低，但如果库存金额占比高，就说明存在库存积压和资金占用风险。

### 第五步：销量预测模型训练与评估

接下来进入“销量预测模型训练与评估”页面。

我按页面步骤操作：

1. 设置预测任务：预测目标为未来 30 天销量 `forecast_30d`。
2. 构建训练样本。
3. 选择训练集 / 测试集划分方式，例如时间序列划分、80/20。
4. 选择特征字段，包括 `sales_30d`、`sales_90d`、`out_freq_90d`、`stock_qty`、`stock_turnover_days`、`gross_profit_rate`、`return_rate`、`abc_class`、`category` 等。
5. 选择预测模型，现场推荐选择 XGBoost 模拟模型。
6. 设置模型参数，例如 `n_estimators`、`max_depth`、`learning_rate` 等。
7. 点击“开始训练并评估模型”。
8. 点击“对比多个预测模型”，查看模型评估结果。
9. 点击“使用推荐模型生成未来销量预测”。

系统会输出：

- `historical_sales_samples.csv`
- `forecast_result.csv`
- `inventory_warning.csv`

这里要强调：第一版为比赛演示版，保留真实建模流程，包括样本构建、特征选择、参数配置、模型评估和预测输出，后续可以替换为真实企业数据和真实训练模型。

### 第六步：补货清仓建议与结果交接

最后进入“补货清仓建议与结果交接”页面，点击“生成补货与清仓建议”。

系统生成：

- `purchase_advice.csv`：补货建议。
- `clearance_advice.csv`：清仓建议。
- `high_priority_products.csv`：高优先级配送商品清单。

其中 `high_priority_products.csv` 会交给模块三，用于识别 A类重点商品、生鲜冷链商品和缺货风险商品的配送优先级。

### 模块二总结

我的模块完成了从商品基础指标到库存智能决策的过程。通过提示词让大模型生成 ABC 分类代码，再结合销量预测和库存预警规则，系统识别出需要补货的重点商品和需要清仓的积压商品，并将高优先级商品清单交给配送规划工程师。

## 模块三一句话定位

模块三接收模块一的配送和网点基础数据、模块二的高优先级商品数据，完成异常订单诊断、网点异常汇总、风险评分、地图图层分析、K-Means配送片区划分、起降点候选筛选和 AI低空应急航线规划。

## 讲稿与操作

各位评委老师好，我是学生3，岗位是配送智能规划工程师。我的任务是基于前两个模块的结果，识别配送异常，计算网点风险，划分配送优化片区，并为重点异常订单规划低空应急配送航线。

然后点击“接收配送规划数据”。

页面会展示配送记录数量、异常初筛、乡镇网点数量、高优先级商品数量和生鲜冷链超时情况。

### 第二步：异常订单诊断规则配置

进入“异常订单诊断规则配置”页面。

为了衔接模块二结果，我先点击“加载商品基础指标表”。系统读取 `product_base_metrics.csv`，用于识别重点商品、生鲜粮油、农产品和冷链食品。

然后配置异常诊断规则：

- 普通订单异常阈值：48 小时。
- A类重点商品阈值：36 小时。
- 生鲜冷链阈值：24 小时。
- 勾选“叠加高优先级商品识别”。

这样设置的原因是：普通订单可以按常规履约标准判断，但 A类重点商品和生鲜冷链商品对时效更敏感，需要更严格的阈值。

点击“运行异常订单诊断”。

当前系统输出：

- 异常配送订单：312 条。
- A类重点商品超时：46 条。
- 生鲜冷链超时：28 条。
- 超长配送订单：96 条。
- 乡镇偏远超时：72 条。

系统同时生成两张表：

- `delivery_exception_result.csv`：配送异常订单明细。
- `outlet_exception_summary.csv`：网点异常订单汇总。

这里要强调：这一页回答的是“哪些订单异常、异常属于哪种类型、异常集中在哪些网点”。

### 第三步：网点异常汇总与风险评分

接下来进入“网点异常汇总与风险评分”。

这一页不是独立计算风险，而是继承上一页异常诊断结果。系统读取：

- `outlet_exception_summary.csv`：上一页异常订单汇总。
- `outlet_metrics.csv`：模块一生成的网点基础指标。

风险评分分为两部分：

- 异常订单结果权重：60%。
- 网点基础数据权重：40%。

我在页面上分别调节滑动权重。

异常订单结果权重包含：

- 异常订单率：20%。
- A类重点商品超时数：15%。
- 生鲜冷链超时数：10%。
- 超长配送订单数：10%。
- 高优先级商品超时数：5%。

网点基础数据权重包含：

- 平均配送时长：10%。
- 历史超时率：10%。
- 退货率：5%。
- 道路等级风险：10%。
- 服务能力不足：5%。

当页面提示异常订单结果权重为 60%、网点基础数据权重为 40% 后，我点击“确认权重并计算风险分”。

系统生成 `outlet_risk_score.csv`，并展示高风险网点、中风险网点和低风险网点。

这里要强调：这一页把“订单层面的异常”转化为“网点层面的风险”，为后续地图分析和聚类分区提供依据。

### 第四步：配送风险地图图层分析

接下来进入“配送风险地图图层分析”。

地图图层分析的作用是先观察风险空间分布，回答“问题在哪里”。

我可以勾选不同图层：

- 全部网点。
- 高风险网点。
- 乡镇网点。
- A类重点商品超时。
- 生鲜冷链超时。
- 候选起降点。

选择地图样式后，点击“生成图层分析地图”。

通过图层叠加可以看到，配送风险不是随机分布，而是在部分乡镇和边缘区域形成明显聚集。这个观察结果说明有必要继续做 K-Means 聚类，把风险区域划分成可管理的配送优化片区。

### 第五步：K-Means聚类参数配置与评估

进入“K-Means聚类参数配置与评估”。

这一页的作用不是判断单个网点风险，而是把网点划分为几个空间配送片区，方便后续起降点筛选和航线规划。

我先选择聚类特征，现场推荐：

- `lon`
- `lat`
- `order_count_90d`
- `timeout_rate_calc`
- `risk_score`
- `high_priority_order_count`

然后选择测试 K 值范围，例如起始值选择 2，结束值选择 6，标准化选择“开启”。点击“第三步：测试K值范围”。

系统展示不同 K 值的轮廓系数和组内误差 SSE，不再额外增加评价列。

接着在“第四步：选择最终K值”中选择 `K=4`，点击“确认并生成网点聚类结果”。

地图上不同颜色的点代表 K-Means 生成的不同配送优化片区，同一种颜色表示同一个片区，不代表风险等级。当前 K=4 对应地图上四个主要空间区域：北部区域、东北区域、中心城区区域和西南区域。

系统输出 `cluster_result.csv`。

### 第六步：起降点候选规则配置

进入“起降点候选规则配置”。

聚类中心可以作为起降点规划的空间参考，但实际起降点不能只看数学中心，还要考虑道路通达能力、服务能力、覆盖范围和重点商品保障能力。因此这一页从每个聚类片区中筛选候选起降点。

现场建议参数：

- 服务半径：5 公里。
- 道路等级门槛：3。
- 服务能力门槛：3。
- 每个聚类片区候选点数量：2。
- 勾选“优先覆盖高风险网点”。
- 勾选“优先覆盖A类商品订单”。
- 勾选“优先覆盖生鲜冷链订单”。

点击“筛选候选起降点”。

当前演示结果固定为四个片区，每个片区 2 个候选点，共 8 个候选起降点。地图上彩色小点表示聚类网点，半透明大点表示候选起降点，并带有“片区X-候选Y”标签。

系统输出 `launch_candidates.csv`。下一步直接进入 AI低空应急配送航线智能规划系统。

### 第七步：AI低空应急配送航线智能规划系统

进入“AI低空应急配送航线智能规划系统”。

这一页只对高优先级异常订单做低空应急规划，普通订单仍然以地面配送为主。

第一步，我选择低空配送任务类型，例如：

- A类重点商品超时订单。
- 生鲜冷链超时订单。

选择后，页面上方的订单数量会随任务类型变化，展示纳入低空规划订单数、A类重点订单数、生鲜冷链订单数和乡镇高风险订单数。

第二步，我选择一个典型高优先级异常订单。页面会显示该订单的起点、终点、订单类型、商品类别、网点风险和配送时长。

第三步，我设置无人机飞行约束：

- 最大飞行距离：根据无人机续航能力和安全冗余设置。
- 最大飞行时间：根据电池续航和冷链时效设置。
- 最大载重：根据商品重量和机型载荷设置。
- 最低返航电量：用于保证返航安全。
- 天气风险：晴、雨、雾、大风会影响安全评分。
- 优化目标：综合评分最优、时效优先、安全优先或覆盖优先。

第四步，我配置模拟强化学习模型。

State 状态变量选择：

- 起降点位置。
- 目标网点位置。
- 商品优先级。
- 商品类型。
- 网点风险等级。
- 飞行距离。
- 商品重量。
- 天气风险。
- 初始电量。

这些状态变量用于描述当前配送任务所处的环境。

Action 动作空间选择：

- A 最短直达航线。
- B 低风险绕行航线。
- C 多网点覆盖航线。
- R 备用返航航线。

这些动作表示系统可以选择的航线方案。

Reward 奖励函数选择“A类冷链超时订单模板”，并设置权重：

- 时效得分：35%。
- 安全得分：25%。
- 重点商品保障：20%。
- 电量余量：15%。
- 覆盖能力：5%。

这样设置的原因是：当前任务是 A类冷链超时订单，时效和安全最重要，同时要保障重点商品，确保返航电量。

接着设置强化学习参数：

- 学习率 `alpha`：控制每次训练对新结果的吸收速度。
- 折扣因子 `gamma`：控制系统对未来收益的重视程度。
- 探索率 `epsilon`：控制系统尝试不同路线的比例。
- 模拟训练轮次：表示系统进行多少轮航线策略评估。

第五步，点击“生成候选航线”。系统根据当前 State、Action、Reward 和飞行约束生成候选航线。

生成后可以通过单选按钮选择不同航线，查看每条路线的综合奖励分、预计距离、预计时间和可视化航线地图。页面还会展示时效奖励、安全奖励、覆盖奖励、重点商品奖励、电量奖励和综合奖励。

如果点击“AI路径评分”或“执行仿真飞行”，系统会生成 `route_simulation_report.html`。如果点击“生成批量路径分配表”，系统会输出 `route_assignment.csv`。

这里要强调：这一页体现的是“强化学习思想的参数化模拟应用”。学生不是一键生成路线，而是先定义状态变量、动作空间、奖励函数和训练参数，再由系统对候选航线进行评分和推荐。

### 模块三总结

我的模块完成了从配送异常发现到低空应急航线规划的闭环。先识别异常订单，再把异常按网点汇总，结合网点基础条件计算风险分；然后通过地图图层观察风险空间分布，用 K-Means 划分四个配送片区；最后筛选每个片区的候选起降点，并对 A类重点商品和生鲜冷链异常订单进行低空航线规划。

- 对 A类重点商品、生鲜冷链商品和乡镇高风险异常订单优先配送。
- 对高风险乡镇网点增加配送资源或纳入低空应急试点。
